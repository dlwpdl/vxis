# `docs/superpowers/` — Current Planning Context

> Current plan, decision log, reviews, and benchmark notes. Old plans/specs are
> removed once they stop describing the live architecture.

## Subdirectories

- [`plans/`](plans/README.md) — current implementation plan
- [`DECISIONS.md`](DECISIONS.md) — dated durable decisions and invalidation context
- [`benchmarks/`](benchmarks/README.md) — Benchmark captures + scan artifacts
- `reviews/` — current audits and cleanup reports

## Convention

Keep only the latest actionable plan. When the direction changes, write a new
dated plan and record the reason in `DECISIONS.md`.
