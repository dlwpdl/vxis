## Themes

### A. God-file / god-method decomposition (cross-cutting design)
Five separate reviewers flagged the same structural problem: oversized units mixing many responsibilities, which is what hid bug #1 across a 400-line caller/callee gap.
- `ScanLoopDecisionPolicyMixin` — 2826 lines, 5 responsibilities (finish-gate, prioritization, skill routing, agent-graph synthesis, focus control). Split into focused mixins mirroring existing `scan_loop_*.py` siblings. (`scan_loop_decision_policy.py:45`)
- `run()` — 1400-line async god-method, 25+ closure vars, ~12 continue/break paths. Extract `_pre_dispatch_guard` / `_post_dispatch_hook` / `_check_finish_gate`. (`scan_loop_run.py:27`)
- `ScanPipeline.run()` — 527 lines, 11 documented steps that should be private methods. (`scan_pipeline_v2.py:661`)
- `_sync_agent_graph_result_to_branches` — 230-line for-loop body. (`scan_loop_agent_graph.py:277`)
- `scan()` CLI command — ~700 lines including a 110-line injection approval gate poking `display._live` private state. (`cli/main.py:108-800`)
- `test_api_security.py` — 727-line file, 5 independent probes, 16 private fns. (`skills/test_api_security.py`)

These are all **large effort**; sequence them and treat #1's predicate/mutation split as the highest-value first slice since it fixes a correctness bug while decomposing.

### B. Correctness risks (beyond Top 8)
- **Duplicated verify_finding block** between `run()` (493-620) and `_dispatch_report_finding_checked()` (actions 738-824) has already diverged (`baseline_size` handling, `UNCONFIRMED` branch). Route all report_finding through the single checked path. (high)
- **Duplicated finish-evidence validation** in `_finish` vs `_completion_evidence_error` — extract one `_validate_finish_evidence()`. (high)
- **`_evaluate_high_severity_poc` dead `needs_control=True`** ignores its `finding_type` arg → false `weak_poc` rejections for misconfig/disclosure findings. Wire in `_finding_type_needs_control()`. (`finding_tools.py:596-640`)
- **Password-reset-poisoning false positives**: `'reset' in body` fires on any "Password reset email sent" 200. Drop that branch; keep `'evil.com' in body`. (`test_auth_deep.py:227`)
- **8 AttackVector `name_en` fields contain Korean + `|||` separator** → report layer renders Korean/separator in English field. Split into `name_ko`. (`scoring/vectors.py:539-588`)
- **Two incompatible `Severity` enums** (`evidence.schema` INFO vs `models.finding` informational) — `_sev_to_level` misses `"info"`, currently only *accidentally* correct via default-0. (`scan_pipeline_v2.py:451`)
- **DecisionClass Literal defined 3× with divergent members** — a `triage`/`critique` hypothesis fails Pydantic validation, swallowed into `v3_errors`. Canonicalize in one module. (`hypothesis/dag.py:19`)
- **Unguarded `next()`** in `_maybe_auto_link_chain` raises StopIteration if a finding vanishes; add default + warn. (`scan_loop_actions.py:1868`)
- **`_save_snapshot` unhandled OSError** crashes an in-progress scan on transient I/O; wrap best-effort. (`agent_graph_tools.py:1340`)
- **`skill_runner` mutates caller's `params` dict** (`params.pop('url')`) → cache-key mismatch bypasses stuck-loop guard. Use `.get` + comprehension. (`skill_runner.py:256`)
- **Two divergent `_safe_scan_id`** (sanitize vs raise) can make a path that passes one fail the other → swallowed trajectory write. Consolidate to the strict version. (`scan_loop_v3.py:1110`)
- **scan_id second-granularity collision** corrupts audit/KB/score files for concurrent scans; append `uuid4().hex[:6]`. (`scan_pipeline_v2.py:699`)
- **Inconsistent nmap service classification** between `_nmap_service_priority` and `_nmap_service_followup_profile` (jenkins ranks 91 vs branch priority 94; db+control-plane label classified differently). Single `_nmap_service_family()`. (`scan_loop_agent_graph.py:1697`) — *medium confidence; behavioral impact depends on real service labels.*
- *Lower confidence / mostly benign:* doubled tautology guards `wanted_title and wanted_title` (decision_policy:158/160) and duplicate `'sqli'` marker (state:111) — harmless but smell of copy-paste that may mask intended conditions; `_model_role_for_decision_class` silently routes `strategy`/`chain` to DIRECTOR (likely intentional, add explicit branch + debug log).

- **Image-load (base64) logic copy-pasted across 3 provider methods**; **3 out-of-sync model/provider default tables** in `brain.py` (risk: primary call picks a different model than the visible fallback chain). Collapse defaults into hybrid_config.

### D. Performance (per-iteration hot paths)
- **`_emit_control_plane()` does O(N)+O(N²) work, called 13+×/iteration**, recomputing `_suggest_chain_candidates()` (nested findings loop) and `_dag_finish_blocking_branches()`. Cache per-iteration snapshot with a dirty flag. (`scan_loop_actions.py:1025`) (medium)
- **`_branch_expected_yield_score` is O(n·k)**: calls full `_forced_branch_action` per branch and recomputes `_branch_family` up to 9×. Cache family locally; cheap `_has_forced_action_candidate()` boolean. (`scan_loop_decision_policy.py:244`)
- **Redundant `build_context_signature`** computed twice per `think()` step. Compute once, pass down. (`brain.py:993,1056`) (low)
- **`_normalize`/`_base_path` redefined as closures + re-import `re`/`urlparse` on every `report_finding`** (hot dedup path). Hoist to module level. (`finding_tools.py:760-801`)

### E. Robustness / resource management
- **`brain._llm_semaphores` class-dict keyed by `id(loop)` leaks** one entry per scan forever; uses deprecated `get_event_loop()` so the registered semaphore may never be the one used. Use per-instance semaphore + `get_running_loop()`. (`brain.py:115,127`)
- **`_build_standard_chain` mutates `os.environ["OPENAI_API_KEY"]`** as a construction side effect → unexpected provider selection. Move to one-time `_resolve_api_keys()`. (`brain.py:334`)
- **Unbounded list growth**: `review_history` / `attempt_outcomes` grow to 1000+ over a 300-iter scan and are serialized in full (`shared_notes` is already capped at 8). Use `deque(maxlen=200)`. (`scan_loop_state.py:510,514`)
- **`parent.evidence` grows unboundedly** (+1140 chars per proven post-exploit worker), serialized into snapshot/dashboard. Cap to ~2000 chars or a bounded list. (`scan_loop_agent_graph.py:230`)
- **7 deferred imports in `run()` wrapped in bare `except: pass`** hide real failures — e.g. broken `finding_tools` silently empties `reported_components` → infinite re-nudging. Hoist stable imports; reserve silent-swallow for truly optional subsystems with debug logging. (`scan_loop_run.py:31`)
- **`attempt_auth` uses `r.response.json()`** (only skill bypassing the AnalyzedResponse abstraction); breaks on mocks/adapters, masked by `except: data={}`. Use `json.loads(r.text)`. (`skills/attempt_auth.py:277,478`)
- **PTI dossier `allow_unicode=False`** mangles Korean evidence into `\uXXXX` in a human-facing knowledge file. Flip to `True`. (`pti/store.py:64`)
- **SSRF round-slicing uses positional `[:8]/[8:16]/[16:]`** instead of a `rounds` schema; payload reorder/growth silently breaks per-round semantics and can yield empty rounds. Migrate `test_ssrf.json` to rounds schema. (`skills/test_ssrf.py:179-186`)
- **CVSS vector_string hardcoded to 9.8 attack profile for all severities** while base_score is lowered — any vector-string renderer shows "Critical" for a LOW/INFO finding. Per-severity templates or `None`. (`scan_pipeline_v2.py:277`) (low)

### F. v2/v3 parity, dead code & test gaps
- **v3 has no ScanPolicy enforcement path**: chokepoints (permit_pivot/strategy/persist_secret) are wired only to `scan_pipeline_v2`; direct `ScanAgentLoop`/SDK runs have no exploitation ceiling or scope enforcement. Attach policy in `initialize_v3_runtime`. (safety/medium)
- **`TrajectoryRecord.cost_usd`/`latency_ms` always 0** and token counts use char//4 proxy → trajectory dataset unusable for its stated distillation purpose. Sample `get_llm_usage_stats()` deltas + `perf_counter`. (`scan_loop_v3.py:1002`)
- **Standard-path `vxis.scope.AuditLog` is dead** — scope violations in non-P1 scans produce no audit trail (p1 path uses its own). Wire it in or delete + document. (`scope/audit.py`)
- **Dead code**: `_call_claude_subprocess` (unreachable, brain.py:2145), `_tools_used` (written, never read, run.py:98), `NORMALIZERS` registry (30 entries, never consumed, normalizer.py:1546), redundant `_urlparse` re-imports in 4 skills.
- **Test gaps (high value)**: branch yield scorer / finish-gate chain + the #1 mutation side effect (zero dedicated tests); duplicated verify block & probe-output/sticky-hint heuristics (fragile 500/≥3 thresholds, untested); `brain._is_refusal`/`_build_smart_history`/`interpret_probe_result`; agent_graph evidence-quality functions; skill_runner cache escalation (the core anti-loop mechanism); pure skill logic (`_forge_alg_none`, `_extract_cloud_metadata`, `_redact_cloud_secrets`); `scoring/vectors.py` public API + ID-uniqueness + `|||`-in-`name_en` guard; assert `v3_errors == []` after a full v3 cycle; scope-gate `shell_exec`/`python_exec` bare-IP bypass (pin the documented gap so a partial fix is detected).
- **Architectural coupling (CLI)**: circular back-import (`interactive.py` imports typer commands from `main.py`; `main.py` imports `run_interactive` back) remains. The two divergent `ScanLiveDisplay` implementations were consolidated on 2026-06-19 by deleting `cli/live_display.py` and moving legacy snapshot rendering into `cli/scan_display.py`; a future `cli/actions.py` split would still reduce the import cycle. `typer.Exit` raised inside async helpers is semantically wrong — use domain exceptions, convert at the sync boundary.
